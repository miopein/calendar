# UI Plan

---

## Визуальный стиль

### Общий стиль
- Минимализм, только чёрный и белый
- Строгая геометрия, без мягких форм и лишнего декора
- Интерфейс как в референсе: высокий контраст, простые линии, акцент на читаемость

### Цвета
- Фон: чёрный
- Основной текст: белый
- Вторичный текст и неактивные элементы: серый
- Акцент для важных сумм и маркеров допускается только в одном дополнительном цвете

### Типографика
- Цифры и время должны выглядеть строго и технично
- Для времени и сумм использовать моноширинный или пиксельный шрифт с чёткими символами
- Крупные значения (часы, суммы, даты) делать контрастными и читаемыми с первого взгляда

### Формы и компоненты
- Прямые углы, без скруглений
- Кнопки и поля с жёсткой рамкой
- Иконки простые, монохромные, без градиентов

### Анимации
- Минимальные и быстрые
- Только функциональные переходы (открытие меню, раскрытие заметки)
- Без декоративных эффектов

## 1. Главный экран

### Шапка
- Иконка гамбургер-меню `≡` слева, кнопка квадратная с жёсткой рамкой
- Заголовок: **TIME TRACKER** (uppercase)
- Справа пустой отступ (spacer), чтобы визуально удерживать заголовок в одной линии

### Верхняя зона
- Строка 1: текущая дата в формате `ДД.ММ.ГГГГ`
- Строка 2: текущее время устройства в формате `ЧЧ:ММ`
- Дата и время обновляются в реальном времени (тик раз в секунду)

---

### Состояния главного экрана

#### Состояние 1 - Idle (ожидание)
- Кнопка по центру: **START**
- Кнопка **SAVE** скрыта
- Блок активной записи в сером/приглушённом состоянии
- Текст активной записи: `--:-- - 0.0h - ...`
- Подпись активной записи: `waiting`

#### Состояние 2 - Running (сессия идёт)
- Кнопка по центру: **STOP**
- Кнопка **SAVE** скрыта
- Блок активной записи становится активным (без приглушения)
- Формат строки: `ЧЧ:ММ - Х.Хh - ...`, где справа не текст, а анимированные 3 точки
- Подпись: `session running`
- Длительность пересчитывается в реальном времени

#### Состояние 3 - Stopped, not saved (остановлено, не сохранено)
- Две кнопки в одной строке:
- **START** слева
- **SAVE** по центру
- Формат строки: `ЧЧ:ММ - Х.Хh - ЧЧ:ММ`
- Подпись: `session stopped, not saved`
- Нажатие **START** возобновляет эту же сессию (исходный start сохраняется, конец снова заменяется на `...`)
- Нажатие **SAVE** открывает модальное окно `SAVE DAY`

#### Состояние 4 - После сохранения
- Сессия сбрасывается в Idle
- Снова показывается `START`
- Активный блок возвращается к `waiting`
- Ниже активного блока показывается только последняя сохранённая запись (одна)

---

### Блок активной записи
- Всегда присутствует под кнопками
- Верхняя строка: интервал и длительность (`ЧЧ:ММ - Х.Хh - .../ЧЧ:ММ`)
- Длительность выводится через общий форматтер (десятичный или `xh ymin`)
- В idle блок визуально приглушён (`inactive`), в running/stopped - обычный

### Блок последней сохранённой записи
- На главной показывается только 1 последняя сохранённая запись
- Формат в 2 строки:
- Строка 1: `ЧЧ:ММ - Х.Хh - ЧЧ:ММ`
- Строка 2: справа дата `ДД.ММ.ГГГГ`
- Комментарий/NOTE в этом блоке не выводится
- Запись скрывается, если длительность больше 16 часов (защитный фильтр в текущей реализации)

### Тексты и лейблы (как в текущем UI)
- Кнопки: `START`, `STOP`, `SAVE`
- Статусы под активной записью: `waiting`, `session running`, `session stopped, not saved`
- Разделитель в строке времени: обычный дефис `-`, не длинное тире

## 2. Модальное окно — Сохранить запись (SAVE DAY)

### Когда открывается
- По кнопке `SAVE` на главном экране (состояние stopped)
- По долгому нажатию `+` в SHIFTS (режим добавления)
- По долгому нажатию `✎` в SHIFTS при выбранной записи (режим редактирования)

### Содержимое

- Заголовок модалки:
- `SAVE DAY` в режиме сохранения
- `EDIT DAY` в режиме редактирования
- Дата (`ДД.ММ.ГГГГ`) - берётся от даты `START`
- Строка `HOURS: ...` с авто-пересчётом длительности
- Поле `START` (один текстовый input)
- Поле `END` (один текстовый input)
- Поле `NOTE` (textarea)

#### Редактирование времени
- Формат времени: строго `HH:MM`
- Инпуты текстовые (`type=text`, `inputmode=numeric`, `maxlength=5`)
- Автонормализация при вводе:
- удаляются нецифровые символы
- после 2 цифр автоматически ставится `:`
- пример: `2309` -> `23:09`
- При полном вводе `START` (5 символов) фокус автоматически переходит в `END`
- `HOURS` пересчитывается сразу при каждом изменении `START/END`

#### Поле заметки
- Заголовок `NOTE`
- Многострочное поле, сохраняется как `note`
- Перед сохранением применяется `trim()`

#### Валидация
- Оба поля `START` и `END` обязательны
- Допустимые значения часов: `00-23`
- Допустимые значения минут: `00-59`
- `END` должен быть строго позже `START` в пределах одного дня
- Если время некорректно: alert `Укажи корректное время START и END`
- Если `END <= START`: alert `END должен быть позже START`
- При невалидных значениях `HOURS` визуально показывает `0.0h` (или эквивалент в текущем формате)

### Кнопки

| Кнопка  | Действие |
|---------|----------|
| **CANCL** | Закрывает модалку без сохранения изменений; возвращает текущий экран/состояние. |
| **SAVE**  | Валидирует поля и сохраняет запись. |

### Поведение сохранения
- В режиме `SAVE DAY` создаётся новая запись `{id, startMs, endMs, note, savedAt}`
- После сохранения из Home:
- сессия сбрасывается в `idle`
- `latestSavedId` обновляется на новую запись
- В режиме `EDIT DAY` изменяется существующая запись, её `savedAt` обновляется, `latestSavedId` указывает на неё
- Данные сохраняются в `localStorage`
- После сохранения модалка закрывается, интерфейс полностью перерисовывается

### Визуальное оформление (текущая реализация)
- Модалка перекрывает весь экран (`position: absolute; inset: 0`) затемнённым фоном
- Центрированный `modal-card` с жёсткой 2px рамкой
- Кнопки `CANCL/SAVE` расположены в одном ряду
- Скругления отсутствуют

## 2. Боковое меню

### Доступность
- Открывается по нажатию иконки гамбургер (≡) в шапке
- Работает из любого состояния приложения
- Закрывается по нажатию на пункт или за пределами меню

### Ограничения в режиме редактирования/сохранения
Когда открыта модалька сохранения или редактирования записи:
- ❌ Недоступны пункты навигации (CALENDAR, SETTING)
- ✅ Доступны только переключатели: TIME FORMAT и иконка луны (смена темы)
- Это предотвращает случайный переход на другой экран во время редактирования

### Содержимое

#### Навигация
- **TIME TRACKER** — возврат на главный экран (главное состояние приложения)
- **CALENDAR** — переход на экран календаря
- **SETTING** — переход на экран настроек

#### Переключатели (внизу меню)

| Элемент | Функция |
|---------|---------|
| **TIME FORMAT** + ползунок | Основной быстрый переключатель формата времени:  <br> **Влево** → `6.8h` (десятичный формат)  <br> **Вправо** → `6h 48min` (часы и минуты)  <br> Формат применяется ко всему приложению (главный экран, логи, календарь) |
| **Иконка луны** 🌙 | Переключение темы:  <br> **Тёмная тема** → чёрный фон, белый текст  <br> **Светлая тема** → белый фон, чёрный текст  <br> Выбор сохраняется локально |

## 3. Календарь

### Архитектура календаря

Календарь реализуется как один экран (`data-screen="calendar"`) с четырьмя вложенными окнами (`.calendar-window`), которые показываются/скрываются через атрибут `hidden`:
- `calendarWindowMenu` — меню выбора (YEAR, MONTH, SHIFTS)
- `calendarWindowYears` — список лет
- `calendarWindowMonths` — список месяцев
- `calendarWindowShifts` — список смен

Переключение между окнами происходит через функцию `showCalendarWindow(level)`, которая управляет атрибутом `hidden`.

### Экран меню календаря

**Содержание меню:**
- Шапка с кнопкой меню (≡) и заголовком "CALENDAR"
- Три кнопки: **YEAR**, **MONTH**, **SHIFTS** 
- Строка "TODAY: ДД.ММ.ГГГГ" внизу

**Структура элементов в коде:**
```
#calendarWindowMenu.calendar-window
├── header.calendar-window-head
│   ├── .window-mark (визуальный элемент ≡)
│   └── h2.screen-title (CALENDAR)
└── div.calendar-menu
    ├── button#openYearsButton.cal-link (YEAR)
    ├── button#openMonthsButton.cal-link (MONTH)
    ├── button#openShiftsButton.cal-link (SHIFTS)
    └── p#calendarToday.calendar-today (TODAY: дата)
```

**Стилизация меню:**
- Контейнер `.calendar-menu`: display grid с gap 0
- Кнопки `.cal-link`: font-size 40px, padding 8px 10px, border-bottom 2px solid
- Текст "TODAY": font-size 20px, margin 10px 8px 0, color muted (серый)
- Шапка `.calendar-window-head`: gap 6px, border-bottom 2px, padding 4px 6px

**Логика кликов:**
- Нажатие YEAR → `state.ui.calendarLevel = "years"` → `renderCalendar()` → скрытие меню, показ окна с годами
- Нажатие MONTH → `state.ui.calendarLevel = "months"` → `renderCalendar()` → скрытие меню, показ окна с месяцами
- Нажатие SHIFTS → `state.ui.calendarLevel = "shifts"` → `renderCalendar()` → скрытие меню, показ окна со смен

**TODAY обновляется автоматически:**
- Функция `renderCalendarMenu()` устанавливает текст в формате "TODAY: ДД.ММ.ГГГГ" с текущей датой

### Переход YEAR -> список годов
- Нажатие **YEAR** открывает экран годов
- На экране годов стрелка **влево** возвращает на экран меню календаря
- Каждая строка года кликабельна как кнопка (нажатие по году открывает месяцы этого года)
- У каждого года есть отдельная кнопка времени (стрелка **вниз**): раскрывает суммарное время за выбранный год
- При первом входе в раздел YEAR автоматически раскрыт текущий год (показана его сумма)

### Переход Год -> список месяцев
- Нажатие на строку года открывает список месяцев этого года
- Каждая строка месяца тоже кликабельна как кнопка (нажатие по месяцу открывает экран SHIFTS для этого месяца)
- У текущего месяца по умолчанию раскрыта сумма за месяц
- Любой другой месяц можно раскрыть вручную отдельной кнопкой времени (стрелкой вниз)
- Стрелка **влево** на экране месяцев возвращает на уровень выше, то есть к списку годов

### Переход SHIFTS -> записи смен
- Нажатие **SHIFTS** открывает текущий месяц со списком смен
- Вверху две кнопки навигации по месяцам:
  - Стрелка **влево**: переход на предыдущий месяц (например, с марта на февраль)
  - Стрелка **вправо**: переход на следующий месяц (например, с марта на апрель)
- В шапке показывается суммарное время за текущий отображаемый месяц

### Поведение записей смен
- У каждой записи показывается интервал времени и длительность
- Кнопка **вниз** у записи появляется только если у записи есть заметка
- Нажатие кнопки вниз раскрывает заметку выпадающим блоком под записью

### Кнопки добавления и редактирования
- Кнопки **плюс** (добавить) и **перо** (редактировать) работают только по долгому нажатию
- После долгого нажатия выполняется переход:
  - **плюс** -> экран ADD DAY
  - **перо** -> экран EDIT DAY

## 4. Настройки

### Экран SETTING
- Заголовок: **SETTING**
- Переход сюда из бокового меню по кнопке **SETTING**
- Стрелка **влево** в шапке возвращает на предыдущий экран

### Пункты настроек

#### Тема
- Переключатель темы (иконка луны):
  - Тёмная тема
  - Светлая тема
- После переключения тема применяется сразу ко всем экранам

#### Экспорт и импорт данных
- Кнопка **EXPORT DATA**: выгружает все записи трекера (дата, время начала, время конца, длительность, заметка)
- Формат экспорта: JSON (базовый) с возможностью расширения до CSV
- Кнопка **IMPORT DATA**: позволяет выбрать файл и загрузить данные обратно в приложение
- Перед импортом показывается подтверждение: заменить существующие записи или объединить с текущими
- После успешного импорта интерфейс календаря и главного экрана обновляется сразу
- При ошибке формата файла показывается сообщение: импорт не выполнен

### Сохранение настроек
- Выбранные значения сохраняются локально на устройстве (localStorage)
- При следующем открытии приложения применяются последние выбранные настройки

### Ограничения при редактировании записи
- Если открыта форма **ADD DAY** или **EDIT DAY**, навигация на другие разделы недоступна
- Разрешено менять только формат времени и тему
