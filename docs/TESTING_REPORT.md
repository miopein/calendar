# Testing Report

Дата: 2026-06-14

## Что тестировалось

1. Базовый цикл сессии:
   - start -> stop -> list
2. Валидация активной сессии:
   - запрет второй активной сессии
   - ошибка stop без активной сессии
3. Валидация редактирования:
   - запрет end_time раньше start_time
   - запись аудита изменений в session_edits
4. Отчеты и фильтры:
   - summary по месяцу
   - фильтр по note_contains
5. Календарь и выгрузка:
   - calendar --month
   - export в JSON и CSV
6. Таймзона и хранение времени:
   - ввод дат со смещением (+0300)
   - проверка нормализации в UTC в БД
7. Ввод вне допустимых рамок и ошибки пользователя:
   - несуществующая дата (например 2026-02-30)
   - некорректный месяц для calendar (например 2026-13)
   - недопустимый limit (0 и отрицательные)
   - конфликтующие фильтры (--day вместе с --month)
   - неполный диапазон (--from без --to)
   - некорректное время (например 99:99)
   - очень большой несуществующий id при edit
   - property-like тесты на случайные невалидные datetime-строки
   - property-like тесты на конфликтные комбинации фильтров и невалидные limits
8. Надежность заметок:
   - заметки с кавычками, спецсимволами и символами LIKE (% и _)
   - очень длинные заметки
   - заметки с переносами строк и табами
   - проверка сохранения/чтения заметок через list и export JSON

## Как запускались тесты

1. Команда:
   - python3 -m unittest discover -s tests -p "test_*.py" -v
