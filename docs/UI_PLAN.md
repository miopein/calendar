# UI Plan

---

## Визуальный стиль

### Общий стиль
- Минимализм, только чёрный и белый
- Строгая геометрия, без мягких форм и лишнего декора
- Интерфейс как в референсе: высокий контраст, простые линии, акцент на читаемость

### Цвета
- Фон: чёрный
- Основной текст: белый
- Вторичный текст и неактивные элементы: серый
- Акцент для важных сумм и маркеров допускается только в одном дополнительном цвете

### Типографика
- Цифры и время должны выглядеть строго и технично
- Для времени и сумм использовать моноширинный или пиксельный шрифт с чёткими символами
- Крупные значения (часы, суммы, даты) делать контрастными и читаемыми с первого взгляда

### Формы и компоненты
- Прямые углы, без скруглений
- Кнопки и поля с жёсткой рамкой
- Иконки простые, монохромные, без градиентов

### Анимации
- Минимальные и быстрые
- Только функциональные переходы (открытие меню, раскрытие заметки)
- Без декоративных эффектов

## 1. Главный экран

### Шапка
- Иконка гамбургер-меню (слева) — открывает боковое меню
- Название приложения: **TIME TRAKER** (по центру или рядом с иконкой)

### Дата и время старта
- Текущая дата отображается в формате `ДД.ММ.ГГГГ`
- Ниже — время старта активной сессии в формате `ЧЧ:ММ`

---

### Состояния главного экрана

#### Состояние 1 — Ожидание (нет активной сессии, нет несохранённой записи)
- Кнопка **START**
- Под кнопкой — пусто или последняя сохранённая запись (горит серым)

#### Состояние 2 — Сессия идёт
- Показаны дата и текущее время старта
- Кнопка **STOP**
- Под кнопкой — текущий лог в формате: `ЧЧ:ММ – Х.Хh – ...`  
  (начало — накопленные часы — конец пока не определён)

#### Состояние 3 — Сессия остановлена, запись не сохранена
- Две кнопки: **START** и **SAVE**
- Под кнопками — лог в формате: `ЧЧ:ММ – Х.Хh – ЧЧ:ММ`
- **Нажатие START в этом состоянии:**  
  Сессия **продолжается** — время старта остаётся прежним, финальное время (конец) убирается.  
  Лог показывает: `ЧЧ:ММ (исходный старт) – Х.Хh – ...`  
  Таймер снова идёт до следующего нажатия STOP.
- **Нажатие SAVE:**  
  Открывается модальное окно «Сохранить запись»

#### Состояние 4 — Запись сохранена
- Лог прошедшей сессии отображается **серым цветом** (не активен, только для просмотра)
- На следующий день сохранённая запись **не отображается** на главном экране
- Кнопка **START** — для новой сессии

---

### Лог сессий под кнопками
- Формат строки: `ЧЧ:ММ – Х.Хh – ЧЧ:ММ`
  - Первый элемент — время старта
  - Второй — накопленная длительность
  - Третий — время финиша (или `...` если сессия ещё идёт)
- Одновременно показывается только текущий день
- Сохранённые записи горят серым

## 2. Модальное окно — Сохранить запись (SAVE DAY)

### Открывается по нажатию SAVE на главном экране

### Содержимое

- **Иконка документа + заголовок SAVE DAY**
- **Дата** — день, за который сохраняется запись (формат `ДД.ММ.ГГГГ`)
- **Hours: Х.Хh** — общее время сессии, пересчитывается автоматически в зависимости от разницы между выбранным началом и концом

#### Редактирование времени
- Четыре поля с прокруткой колёсика мыши (или свайпом на мобиле):
  - Часы старта — Минуты старта → Часы финиша — Минуты финиша
  - Формат: `ЧЧ ▼ | ММ ▼ — ЧЧ ▼ | ММ ▼`
- При изменении любого поля `Hours: Х.Хh` пересчитывается мгновенно

#### Поле заметки
- Заголовок **NOTE**
- Многострочное текстовое поле для произвольного текста

### Кнопки

| Кнопка  | Действие |
|---------|----------|
| **CANCL** | Закрыть модалку, вернуться на главный экран. Несохранённая запись остаётся в состоянии 3 (START + SAVE). |
| **SAVE**  | Сохранить запись в базу. Вернуться на главный экран. Запись отображается серым. Через сутки эта запись убираеться из главного экрана|

## 2. Боковое меню

### Доступность
- Открывается по нажатию иконки гамбургер (≡) в шапке
- Работает из любого состояния приложения
- Закрывается по нажатию на пункт или за пределами меню

### Ограничения в режиме редактирования/сохранения
Когда открыта модалька сохранения или редактирования записи:
- ❌ Недоступны пункты навигации (CALENDAR, SETTING)
- ✅ Доступны только переключатели: TIME FORMAT и иконка луны (смена темы)
- Это предотвращает случайный переход на другой экран во время редактирования

### Содержимое

#### Навигация
- **TIME TRACKER** — возврат на главный экран (главное состояние приложения)
- **CALENDAR** — переход на экран календаря
- **SETTING** — переход на экран настроек

#### Переключатели (внизу меню)

| Элемент | Функция |
|---------|---------|
| **TIME FORMAT** + ползунок | Основной быстрый переключатель формата времени:  <br> **Влево** → `6.8h` (десятичный формат)  <br> **Вправо** → `6h 48min` (часы и минуты)  <br> Формат применяется ко всему приложению (главный экран, логи, календарь) |
| **Иконка луны** 🌙 | Переключение темы:  <br> **Тёмная тема** → чёрный фон, белый текст  <br> **Светлая тема** → белый фон, чёрный текст  <br> Выбор сохраняется локально |

## 3. Календарь

### Экран меню календаря
- Пункты: **YEAR**, **MONTH**, **SHIFTS**
- Кнопка меню (гамбургер) открывает боковое меню

### Переход YEAR -> список годов
- Нажатие **YEAR** открывает экран годов
- На экране годов стрелка **влево** возвращает на экран меню календаря
- Каждая строка года кликабельна как кнопка (нажатие по году открывает месяцы этого года)
- У каждого года есть отдельная кнопка времени (стрелка **вниз**): раскрывает суммарное время за выбранный год
- При первом входе в раздел YEAR автоматически раскрыт текущий год (показана его сумма)

### Переход Год -> список месяцев
- Нажатие на строку года открывает список месяцев этого года
- Каждая строка месяца тоже кликабельна как кнопка (нажатие по месяцу открывает экран SHIFTS для этого месяца)
- У текущего месяца по умолчанию раскрыта сумма за месяц
- Любой другой месяц можно раскрыть вручную отдельной кнопкой времени (стрелкой вниз)
- Стрелка **влево** на экране месяцев возвращает на уровень выше, то есть к списку годов

### Переход SHIFTS -> записи смен
- Нажатие **SHIFTS** открывает текущий месяц со списком смен
- Вверху две кнопки навигации по месяцам:
  - Стрелка **влево**: переход на предыдущий месяц (например, с марта на февраль)
  - Стрелка **вправо**: переход на следующий месяц (например, с марта на апрель)
- В шапке показывается суммарное время за текущий отображаемый месяц

### Поведение записей смен
- У каждой записи показывается интервал времени и длительность
- Кнопка **вниз** у записи появляется только если у записи есть заметка
- Нажатие кнопки вниз раскрывает заметку выпадающим блоком под записью

### Кнопки добавления и редактирования
- Кнопки **плюс** (добавить) и **перо** (редактировать) работают только по долгому нажатию
- После долгого нажатия выполняется переход:
  - **плюс** -> экран ADD DAY
  - **перо** -> экран EDIT DAY

## 4. Настройки

### Экран SETTING
- Заголовок: **SETTING**
- Переход сюда из бокового меню по кнопке **SETTING**
- Стрелка **влево** в шапке возвращает на предыдущий экран

### Пункты настроек

#### Тема
- Переключатель темы (иконка луны):
  - Тёмная тема
  - Светлая тема
- После переключения тема применяется сразу ко всем экранам

#### Экспорт и импорт данных
- Кнопка **EXPORT DATA**: выгружает все записи трекера (дата, время начала, время конца, длительность, заметка)
- Формат экспорта: JSON (базовый) с возможностью расширения до CSV
- Кнопка **IMPORT DATA**: позволяет выбрать файл и загрузить данные обратно в приложение
- Перед импортом показывается подтверждение: заменить существующие записи или объединить с текущими
- После успешного импорта интерфейс календаря и главного экрана обновляется сразу
- При ошибке формата файла показывается сообщение: импорт не выполнен

### Сохранение настроек
- Выбранные значения сохраняются локально на устройстве (localStorage)
- При следующем открытии приложения применяются последние выбранные настройки

### Ограничения при редактировании записи
- Если открыта форма **ADD DAY** или **EDIT DAY**, навигация на другие разделы недоступна
- Разрешено менять только формат времени и тему
